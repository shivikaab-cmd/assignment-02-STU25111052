This assignment is a comprehensive exploration of modern data systems and how different database paradigms are used to solve real-world problems. It covers a wide spectrum of technologies ranging from traditional relational databases to cutting-edge vector databases used in AI applications.
Initially, SQL-based systems appeared sufficient for most data-related tasks. However, as the assignment progressed, it became evident that different use cases demand specialized storage and processing systems. Each section of this project highlights how the choice of database architecture directly impacts scalability, performance, and flexibility.
Through this assignment, I gained a deeper understanding of how data flows across systems — from structured transactional storage to analytical processing, and finally to intelligent querying using embeddings.

Key Learnings:
Importance of database normalization (3NF) in eliminating anomalies
Differences between RDBMS and NoSQL systems
Designing scalable data warehouse architectures (Star Schema)
Working with semi-structured & unstructured data in data lakes
Using DuckDB for cross-format querying (CSV, JSON, Parquet)
Understanding vector embeddings & semantic similarity
Building an end-to-end data architecture system

Skills Demonstrated:
SQL Query Writing & Optimization
Data Modeling & Schema Design
ETL Thinking & Data Cleaning
NoSQL Data Structuring (MongoDB)
Analytical Querying (OLAP)
Python for Machine Learning Concepts
System Design & Architecture Thinking

Project Highlights - 
This repository demonstrates:

Conclusion:
This assignment strengthened my understanding of how modern data ecosystems operate. It clearly demonstrated that no single database solution fits all use cases, and choosing the right tool depends on the nature of the data and the problem being solved.

Overall, this project provided a strong foundation in data engineering concepts and prepared me for tackling real-world data challenges with confidence.
